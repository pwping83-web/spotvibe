# 스팟바이브(SpotVibe) — 프로젝트 개요

## 한 줄 정의

**실시간·지역 맥락을 바탕으로 “지금 이 근처에서 무엇이 일어나는지”를 지도와 이벤트·알림으로 보여 주고, AI 스타일의 행동 지시로 이동·타이밍·무드를 돕는 모바일 웹 앱 프로토타입.**

브랜드 표기는 화면에서 **스팟바이브**로 통일하는 방향입니다.

---

## 컨셉

1. **핫스팟(Hotspot) + 바이브(Vibe)**  
   특정 구역(예: 홍대·연남 일대)에서 연령대·이벤트·군집(클러스터)으로 “활기”를 시각화합니다.

2. **탐색 중심**  
   사용자가 관심 지점을 바꾸면 히트맵·점 분포·이벤트 핀이 같은 기준으로 이동해, “내가 보는 중심”과 데이터가 맞춰집니다.

3. **행동 지시형 AI UI**  
   지도에서 클러스터 등을 탭하면 하단 시트가 열리고, 이동 수단(자차 / 킥보드 / 뚜벅이·따릉이)에 맞는 경로 안내 문구, 미니맵, 액션 카드, 외부 지도(네이버 검색 링크)를 제공합니다.

4. **마이 페이지 프로필**  
   선호 시간대, AI 추천 on/off, 카카오 연동 UI(모의), **이동 프로필(MobilityProfile)** 이 BottomSheet의 문구와 연결됩니다.

---

## 수익 구조 (기획 방향)

**핵심:** 이용자가 실제로 스크롤·탭하는 **이벤트 탭**에 **광고·유료 노출 구역**을 두는 모델입니다.  
당근마켓의 **지역(동네) 기반 광고**처럼, **“사람이 보는 곳”** 에 지역 행사·버스킹·페스티벌·매장 프로모션 등을 **유료 리스팅**으로 올리는 구조를 전제로 합니다.

| 요소 | 설명 |
|------|------|
| 인벤토리 | 이벤트 리스트 상단·피드 내 **파트너 슬롯**, 필터와 섞이지 않게 **광고 라벨**(예: 파트너·광고)로 식별 |
| 과금 단위 | 지역(반경·행정동)·기간·노출 순위(CPC/CPM/고정 월비 등은 사업·법무와 별도 확정) |
| 광고주 | 지역 상권, 공연·행사 주최, 브랜드 로드쇼, B2B 파트너 |
| 사용자 경험 | 자연스러운 “이벤트” 형태로 노출하되 **스폰서 표시**로 투명성 유지(표시광고법·플랫폼 정책 검토 필요) |

지도 탭의 핀·클러스터와 **연동한 유료 강조**는 2차 확장으로 둘 수 있습니다. 현재 UI의 **이벤트·파트너 표기**는 이 수익 모델을 염두에 둔 자리입니다.

## 주요 화면·기능 (현재 구현 기준)

| 영역 | 설명 |
|------|------|
| 로그인 | 클라이언트만의 간단 인증 상태; 실제 OAuth는 TODO로 남김 |
| 지도 | Leaflet + Carto Dark Matter, 내 위치(GPS), 연령대 점(투명 군집), 이벤트 핀, 지역 프리셋 빠른 이동 |
| **AI 추천 버튼** | 지도 우하단 버튼 탭 → 마이 프로필 기반 **개인화 추천 카드**(30초 카운트다운, 다음 버튼으로 순환) |
| **AI 인도 말풍선** | 지도를 보는 중 **가끔(30~60초 간격)** 저절로 나타나는 「여기 어때? / 여기는?」 말풍선 + 화살표·점선 경로 |
| **심장 박동 오버레이** | MBTI·성별·활동 구역 원이 심장처럼 살짝 맥박 치는 애니메이션 |
| **내 위치 / 탐색 중심** | GPS 켜면 파란 점 + 「내 위치」, GPS 없으면 탐색 중심에 「탐색 중심」 라벨 |
| 이벤트 | 리스트·필터 칩·**파트너·광고** 표기(→ 유료 지역·행사 노출 구조와 연결) |
| 알림 | 타임라인형 알림, 무드 이미지 |
| 마이 | MBTI·혈액형·활동 태그·성별 인파·이동 프로필·선호 시간 설정 |
| **운영·문의** | 마이 탭 하단 — 개선 제안·광고·행사·기타를 메일 앱 연동으로 접수 |
| **헤더** | 스팟바이브 ← 실시간 시계 → 현재 탭 표시, 항상 최상단 고정 |

## 타깃 사용자(가정)

- 라이브·버스킹·야외 모임 등 **“지금 어디가 붐비는지”** 를 빠르게 보고 싶은 이용자.
- 대중교통·PM·자차 등 **이동 방식이 다른 사람**에게 맞춘 한 줄 안내를 원하는 이용자.
