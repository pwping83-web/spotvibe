# 즉시 실행 체크리스트 (비상·징후 대응)

아래는 **가능한 한 서비스 중단 시간을 짧게** 유지하면서, **우선순위대로** 실행하기 위한 목록입니다.

---

## A. 키·비밀 유출이 의심될 때 (최우선)

- [ ] **GitHub/GitLab 등** 저장소·이슈·Actions 로그에 키가 올라간 적 없는지 검색 (`service_role`, `SUPABASE_SERVICE`, `eyJ` JWT 패턴 등).
- [ ] **Supabase Dashboard** → Project Settings → API  
  - `anon` 키만 클라이언트 번들에 포함되는지 재확인.  
  - **`service_role`이 프론트·환경 변수(브라우저 노출)·펌웨어에 없는지** 즉시 확인.
- [ ] 유출 의심이 크면 **Service Role 키 회전**(재발급) 후, **Edge Function·CI·로컬 `.env`** 전부 교체 배포.  
  - 회전 직후 **짧은 5xx/인증 오류**가 날 수 있음 → 배포 파이프라인부터 맞춤.
- [ ] 유출 경로가 Git이면 **해당 커밋 히스토리 정리**(BFG/git filter-repo 등) — 조직 정책에 따름.

**기능 영향**: 키만 정확히 교체하면 일반 사용자는 거의 무감. 잘못된 키를 넣으면 전체 장애이므로 **스테이징에서 먼저 검증** 권장.

---

## B. 트래픽·에러율이 비정상일 때 (DDoS·스크래핑 의심)

- [ ] **호스팅 대시보드**(Netlify/Vercel/Cloudflare 등 사용 시)에서 요청 수·5xx·지연 스파이크 확인.
- [ ] **Supabase** 대시보드에서 API 요청·DB CPU·동시 연결 확인.
- [ ] 정적 사이트 앞단에 **CDN/WAF**가 있으면:  
  - 국가/ASN 차단은 **오탐(해외 정상 사용자)** 위험 → 가능하면 **레이트리밗·챌린지**부터.
- [ ] **특정 경로만** 공격이면 해당 경로에만 제한(Edge Function 앞단, 또는 호스팅 규칙).

**기능 영향**: 전 세계 차단은 금지 수준으로 신중히. **IP·경로 단위**가 사용자 영향이 적음.

---

## C. 계정 탈취·악성 행위가 보일 때

- [ ] Supabase **Auth**에서 해당 사용자 세션 무효화·비밀번호 재설정(정책에 따라).
- [ ] DB **`user_moderation`** 등 운영 도구로 계정 상태 확인·일시 정지(이미 마이그레이션에 정의된 경우).
- [ ] **SOS·제보 스팸**은 기존 **일일 한도·신고·관리자 RPC** 흐름으로 우선 차단.

**기능 영향**: 오탐 시 정상 유저 불편 → **관리자 복구 절차**를 짧게 문서화.

---

## D. 릴리스 직전 (5분 버전)

- [ ] 프로덕션 빌드에 **디버그 로그·콘솔에 토큰 출력** 없음.
- [ ] `.env` / `import.meta.env` — **브라우저에 노출되는 변수만** `VITE_` 등 설계대로.
- [ ] Storage 버킷 **공개/비공개** 정책이 의도와 일치하는지 대시보드에서 재확인.
