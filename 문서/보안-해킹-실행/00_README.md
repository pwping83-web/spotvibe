## SpotVibe 스택에 맞춘 요지

- **프론트**: Vite + React (`문서/제품-기획/04_기술-구조.md` 참고)  
- **백엔드**: Supabase (Auth, PostgreSQL **RLS**, Edge Functions, Storage)  
- **이미 코드·DB에 있는 남용 방지 예**: SOS **일 1회** 한도 트리거, 신고·제재 테이블 (`supabase/migrations/` 내 SOS·모더레이션 관련 파일)

## “기능 이상 없이” 방어를 올릴 때 원칙

1. **저위험 → 고위험 순**: 모니터링·알림 → 키 회전 → WAF/레이트리밗 → RLS·스키마 변경  
2. **한 번에 하나씩**: 동시에 여러 제한을 켜면 정상 유저 오탐이 늘어납니다.  
3. **측정 후 조정**: 레이트리밗·WAF 규칙은 로그를 보고 임계값을 완화합니다.

## 운영 일반(끊김·느림·용량)과의 구분

**서비스 다운·버벅임·사진 저장소 부족·플랜 한도** 등은 `문서/유저증가-운영-대응-실행/` 폴더를 먼저 보세요. 이 폴더는 **악의적 공격·비밀 유출·웹 취약점**에 초점을 둡니다.

## 문서 갱신

Supabase·호스팅사 UI가 바뀔 수 있으므로, **대시보드 메뉴명은 공식 문서를 함께 확인**하고 이 폴더의 절차를 주기적으로 맞춰 주세요.
