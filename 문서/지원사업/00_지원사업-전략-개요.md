# SpotVibe(지금여기) — 정부지원사업 전략 개요

> 작성일: 2026-05-08  
> 목적: 스마트시티·재난안전·ICT R&D·창업 지원사업 제출을 위한 사업 포지셔닝 및 전략 정리

---

## 1. 핵심 포지셔닝 (심사위원 언어로)

**SpotVibe는 "유령 앱 문제"를 해결하는 소프트웨어 기반 능동형 시민 안전망입니다.**

| 심사 관점 | 포지션 |
|-----------|--------|
| 사회 문제 | 골든타임(4~5분) 내 공공 대응 불가 → 시민 초기 대응망 공백 |
| 기술 차별성 | 일상 DAU 확보 기반 SOS 대기자 상시 확보 (유령 앱 현상 구조적 해결) |
| 공공 기여 | 지자체 재난 관제 예산 절감, 하드웨어 없는 스마트시티 안전망 |
| 확장성 | B2C 무료 앱 → B2G SaaS 관제 대시보드 구독 수익 |
| 실증 가능성 | Supabase·Leaflet·Edge Functions 기반 MVP 완성, 즉시 실증 투입 가능 |

---

## 2. 사업 유형별 공략 방향

### 🏙️ Track A — 스마트시티 실증형 (국토교통부·KAIA)
- **핵심 메시지**: "전 국민 스마트폰을 CCTV·재난 센서로 전환하는 하이퍼로컬 관제 소프트웨어"
- **강점**: 하드웨어 불필요, 지자체 예산 절감, 즉시 실증 가능
- **제출 포인트**: AI 검증 현장 사진 + 실시간 LBS 군집 데이터 → 지자체 대시보드 연동

### 🚨 Track B — 재난안전 R&D (행정안전부·IRIS)
- **핵심 메시지**: "골든타임 내 이웃 초기 대응을 체계화한 시민참여형 재난 보조망"
- **강점**: 공식 119/112와 역할 분리(보조), 허위 신고 다층 방지, 실증 데이터 즉시 생성
- **제출 포인트**: 4유형 SOS(화재·치안·조난·구급) + AI 검증 + 일일 1회 한도 + KST 기반 운영

### 💡 Track C — ICT R&D / 디지털 안전 (과기정통부·NIPA)
- **핵심 메시지**: "AI·LBS 융합 시민 안전 플랫폼 — 디지털 안전 선도모델"
- **강점**: EXIF + 비전 LLM + 텍스트 LLM 다층 모더레이션, Edge Functions 기반 경량 AI
- **제출 포인트**: verify-sos-photo Edge AI + 실시간 Realtime 구독 기술

### 🚀 Track D — 창업 성장 지원 (중기부·창업진흥원)
- **핵심 메시지**: "소셜 임팩트 플랫폼 스타트업 — 안전망 + 일상 플랫폼 융합 BM"
- **강점**: MVP 완성, 즉시 사용자 모집 가능, 소셜벤처 인증 가능성
- **제출 포인트**: 네트워크 효과 기반 자생 성장 모델, B2G 수익 구조

### 🔵 Track E — 치안·사회안전 (경찰청·폴리스랩)
- **핵심 메시지**: "범죄·위협 상황 시민 신속 알림 및 커뮤니티 초기 대응 기술"
- **강점**: public_safety 유형 SOS, 범죄 연계 의심 신고 카테고리, 관리자 모더레이션
- **제출 포인트**: 치안 유형 SOS + 범죄 연계 신고 + RLS 기반 데이터 보호

## 3. 공통 강조 포인트 (모든 Track)

### ① 유령 앱 현상 해결
기존 SOS 앱은 일상 사용 동기 없음 → 위기 시 수신자 0명. SpotVibe는 **군집 밀도 탐색이라는 일상 기능**으로 상시 DAU를 확보하여 이 문제를 구조적으로 해결합니다.

### ② 다층 신뢰성 확보
- EXIF 무결성 검사 (촬영 시각·Software 태그)
- 비전 LLM 허위 사진 판정 (`verify-sos-photo`)
- 텍스트 LLM 모더레이션 (`moderate-user-content`)
- KST 1일 1회 발신 한도
- 다인 신고 → 자동 `user_moderation` 연동
- 관리자 모더레이션 UI

### ③ 개인정보 최소 수집 설계
- 위치: 발신자 GPS만 수집, 만료(15분) 후 신호 종료
- 군집 표시: 비식별 집계(연령대·MBTI 태그만), 개인 식별 불가
- 대시보드: 집계·감사 로그 중심, 개인 식별 최소화

### ④ 119/112와의 명확한 역할 분리
"공식 응급기관의 대체가 아닌 이웃 초기 대응 보조"를 UI·약관·교육 콘텐츠에 고정. 과제 심사에서 법적 책임 리스크 우려를 해소합니다.
