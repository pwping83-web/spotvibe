### 2-2. 인파 사고 예방형 (B2G 설득 핵심)

#### 이태원 압사 사고 유형
- **문제**: 골목 병목 지점에서 인파 정보가 없어 미리 우회 불가
- **SpotVibe 해법**: 현장 제보자들이 올린 실시간 사진·혼잡도 정보 → 진입 전 확인 가능
- **실제 구현**: `spot_reports` 테이블 — 현장 사진 GPS 태그 + 시간 정보 + AI 모더레이션

#### 광화문 집회 / 국가 행사 인파
- **문제**: 지하철 출구에서 나오면 이미 밀집 상태
- **SpotVibe 해법**: 사용자들이 출구별 혼잡도 제보 → 다른 출구 우회 안내
- **수치 논리**: 100명 중 10~20명만 우회해도 병목 밀도 20~40% 감소 (20% 마중물 이론)

#### 야구장 (SSG랜더스 연계 실증 대상)
- **연결 포인트**: SSG랜더스 홈구장(인천SSG랜더스필드) 경기 당일 사용 시나리오
- 구장 주변 인파 밀도 제보 → 주차장 선택, 외부 식당 이동 등 의사결정 지원
- 경기 종료 후 대규모 이동 시 지하철·버스 혼잡도 사전 파악
- **현장 실증 플랜**: 경기 당일 현장 제보 유도 이벤트 + QR 코드 배포

#### 축제·페스티벌 (지역 행사)
- **구현 완료**: `public_notices` 테이블 — 행안부 긴급재난문자 + 공공데이터포털 지역 문화행사 자동 수집
- 축제 행사 정보 자동 연동 → 이벤트 탭에 실시간 표시
- 현장에서 QR → 주변 화장실·소화기·혼잡도 즉시 파악

#### 바닷가·휴양지 인파
- 여름 해수욕장 사망 사고 → 익수자 발생 시 SOS 발신 (구급 유형)
- 해변 화장실 위치 제보 맵 → 즉시 조회 가능
- 주차 혼잡도·이동 동선 제보

#### 지하철 인파 (명절·출퇴근 러시아워)
- 특정 역·호선 혼잡도 현장 제보
- 승강장 안전 관련 SOS 발신 가능 (치안·구급 유형)

## ⑤ 지원금 활용 계획 — 예상 질문 완전 방어

  - 변리사 위임 (특허 보강·우선심사·추가 출원)
  - SSG랜더스 경기 현장 실증 이벤트 진행비

### Q. "3.3억은 너무 많은 거 아닌가요?"

## ⑥ 개인정보·위치정보 질문 방어

### Q. "개인 위치를 불법으로 추적하는 것 아닙니까?"

**A.** 두 가지 레벨로 분리 설명:

**① 군집 밀도 지도 (일반 사용자)**
- 개인을 추적하지 않습니다. 현재 시각의 1분 버킷 값을 난수 시드로 사용하여 **클라이언트에서** 익명 분포 점을 생성합니다.
- 서버에 개인 좌표가 저장되지 않으며, 보이는 것은 연령대별 '비식별화된 분포'뿐입니다.
- GPS를 켜면 내 위치 기준 주변 정보가 나오는 것이고, 이것은 카카오맵·네이버 지도와 동일한 방식입니다.

**② SOS 발신 시**
- 사용자가 **자발적으로** SOS를 발신할 때만 좌표가 서버에 저장됩니다.
- 저장 후 **15분 자동 만료** — 이후 데이터 삭제.
- RLS(Row Level Security) 정책으로 본인·반경 수신자만 접근 가능.
- "위치정보 수집·이용 동의"를 명시적으로 받습니다.

**③ 취약계층 (구청 연계 시)**
- 공공기관 담당자 승인을 통해서만 등록 가능.
- 개인 식별 정보는 RLS로 완전 차단, 좌표만 노출.
- 위치정보사업자 신고는 사업 확장 시 진행 예정.

## ⑦ SSG랜더스 연계 실증 — 차별화 포인트

### 왜 SSG랜더스인가?

| 이유 | 내용 |
|------|------|
| 인파 집중 | 경기 당일 2만~3만 명 집중, 시간대별 병목 명확 |
| 지역 밀착 | 인천 연고 구단 → 인천지식재산센터 지역 사업 연계 적합 |
| QR 배포 용이 | 구장 내 안내판·입장권·전광판 활용 가능 |
| 데이터 수집 | 경기 전·중·후 인파 흐름 실시간 제보 데이터 수집 |

### 실증 시나리오 (현장 운영)

```
[경기 전 - D-2시간]
  전광판·입장권에 QR 코드 노출
  → 팬들 스캔 → "오늘 내야 1루 쪽 사람 많아요" 제보 시작

[경기 중]
  먹을거리 줄 얼마나 긴지 제보
  화장실 혼잡도 제보 → 다른 층 화장실 안내

[경기 후 - 대규모 이동]
  지하철 혼잡도 실시간 제보
  주차장 출구별 정체 현황 제보
  → 이용자들 분산 효과 측정

[데이터 산출]
  제보 건수, 앱 접속자 수, 체류 시간, SOS 발신 여부
  → 지자체 제출용 실증 보고서 작성
```

## ⑧ 어려운 용어 사전 (면담 중 즉시 참조용)

| 용어 | 쉬운 설명 |
|------|---------|
| **PWA** (Progressive Web App) | 앱 설치 없이 웹브라우저에서 앱처럼 작동하는 기술. QR 스캔 후 바로 열림. |
| **Supabase** | 서버 데이터베이스 + 실시간 통신을 제공하는 클라우드 서비스 (AWS 같은 것) |
| **RLS** (Row Level Security) | 데이터베이스에서 "이 사람은 자기 데이터만 볼 수 있다"는 보안 규칙 |
| **PRNG** (Pseudo Random Number Generator) | 같은 시간대에 모든 기기에서 동일한 무작위 숫자를 생성 → 개인 데이터 없이 군중 분포 시각화 가능 |
| **EXIF** | 사진 파일에 자동 저장되는 메타정보 (촬영 시간·위치·기기 등) |
| **LLM** (Large Language Model) | ChatGPT 같은 대형 언어 AI 모델 |
| **DAU** (Daily Active Users) | 하루에 실제 앱을 켜는 사용자 수 |
| **Realtime** | 서버에서 정보가 바뀌면 즉시 내 화면에도 반영되는 기능 |
| **B2G** | 기업이 정부·지자체에 서비스 판매하는 사업 모델 |
| **Edge Function** | 전 세계 가까운 서버에서 실행되는 가벼운 AI 처리 코드 |
| **TRL** (Technology Readiness Level) | 기술성숙도. 1(아이디어)~9(상용화). 현재 4~5 수준 |
| **넛지(Nudge)** | 강제하지 않고 자연스럽게 행동을 유도하는 방식 (쿠폰·추천으로 골목 상권 유도 등) |
| **히트맵** | 사람이 많은 곳을 빨갛게, 적은 곳을 파랗게 표시하는 지도 시각화 |
| **SaaS** | 소프트웨어를 구독료로 제공하는 서비스 형태 (월/연 구독) |

## ⑨ 현장에서 바로 보여줄 시연 순서 (스마트폰)

1. **QR 스캔** → 앱 오픈 (설치 없음 강조)
2. **카카오 로그인** → 1초 진입
3. **지도 화면** → 주변 익명 군집 점 분포 보여주기
4. **화장실 버튼** 탭 → 반경 500m 화장실 마커 표시 + 사진 클릭
5. **소화기 버튼** 탭 → 반경 3km 소화기 위치 표시
6. **SOS 버튼** → 1단계 "119/112 먼저" 안내 화면 보여주기 (실제 발신 안 함)
7. **이벤트 탭** → 행안부 재난문자·지역행사 표시 화면
8. **보호망(🦯) 버튼** → 취약계층 도움 시스템 설명

**시연 멘트**: "지금 보시는 이 모든 기능이 앱 설치 없이 웹에서 작동하고 있습니다."

## ⑩ 최종 보스급 압박 질문 방어

### Q. "앱을 열지 않는 99%의 시민에게 어떤 효과가 있습니까?"

**A.** "100%를 통제할 필요가 없습니다. **20% 마중물 이론**입니다. 이태원 사고 당시 골목 입구에 서 있는 사람 100명 중 20명만 '여기 위험하다'는 정보를 받고 다른 골목으로 이동해도, 나머지 80명에게 가는 물리적 압력이 획기적으로 줄어듭니다. 저희는 앱을 켠 스마트 사용자 20%를 먼저 우회시키는 것이 목표입니다."

### Q. "구현이 정말 됐습니까? 껍데기 아닌가요?"

**A.** "Supabase 마이그레이션 타임라인이 2026년 5월 1일부터 5월 13일까지 **13개 이상의 데이터베이스 변경 이력**으로 남아 있습니다. SOS 4유형 테이블, AI 검증 Edge Function, 집단 신고 자동 처리, 취약계층 메시지 시스템까지 실제 코드와 DB가 존재합니다. 직접 스마트폰으로 시연해 드리겠습니다."

### Q. "경쟁사가 따라 만들면 어쩔 건가요?"

**A.** "특허 출원이 완료되어 있어 핵심 기술 방어가 가능합니다. 또한 네이버·카카오가 저희 방식을 채택한다면 오히려 저희 기술력이 검증된 것이고, 지자체 SaaS 대시보드는 데이터 축적이 있는 선점자가 유리합니다. 첫 번째 실증 지역 데이터는 저희만 가집니다."

### Q. "사용자가 모이지 않으면 군집 데이터가 없는 거 아닌가요?"

**A.** "정확한 지적입니다. 그래서 **PRNG 기반 익명 군집 시뮬레이션**이 핵심 설계입니다. 실제 사용자가 0명이어도 시각적으로 사람이 있는 것처럼 보이는 것이 아니라, 실제 사용자가 없어도 서비스가 동작하면서 사용자를 모을 수 있는 '닭이 먼저냐 달걀이 먼저냐' 문제를 해결하는 방법입니다. 실증 기간 중 SSG랜더스 현장 QR 배포로 초기 사용자를 확보합니다."

### Q. "정부 예산으로 특허 비용을 쓰는 게 맞습니까?"

**A.** "중소기업·스타트업 지원사업에서 IP 강화 예산은 허용 항목입니다. 특히 지식재산센터 연계 사업은 IP 경쟁력 확보가 사업 목적 중 하나입니다. 이미 자비로 출원을 완료했고, 지원금으로는 변리사 청구항 보강·우선심사·PCT 국제 출원 등 고도화 절차를 진행할 계획입니다."
