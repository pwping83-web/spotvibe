# SpotVibe — 정부지원사업 사업계획서 공통 핵심항목

> 작성일: 2026-05-08  
> 용도: 재난안전·스마트시티·ICT R&D·창업 지원사업 신청 시 공통으로 쓸 수 있는 항목별 초안  
> 주의: 각 공모 요강에 맞게 항목 순서·분량·강조점을 반드시 조정할 것

---

## 1. 사업 개요 (1~2페이지)

### 1.1 서비스명 및 비전
**서비스명**: 지금여기 (SpotVibe)  
**부제**: 골든타임을 사수하는 하이퍼로컬 시민 초기 대응 플랫폼  
**비전**: 일상 앱의 DAU를 사회적 안전망으로 전환하는 소프트웨어 기반 능동형 구조망 구축

### 1.2 핵심 문제 정의

**문제 ①: 골든타임 내 공공 대응의 물리적 한계**
- 119 신고 후 현장 도착까지 평균 7~10분 소요
- 심정지 4분, 화재 초기 1분 — 이 시간을 메울 공공 인프라 없음

**문제 ②: 기존 SOS 앱의 구조적 실패 (유령 앱 현상)**
- 안전·위기 목적으로만 설계된 앱은 평상시 사용 동기 없음
- 위기 발생 시 주변에 앱을 켜고 있는 수신자 0명 → 구조 불가

**문제 ③: 고비용 하드웨어 관제의 사각지대**
- 스마트 CCTV·비상벨 등 고비용 하드웨어는 골목·사각지대 미커버
- 지자체 스마트시티 예산의 상당 부분이 하드웨어에 소모

**문제 ④: 취약계층(시각장애인·이동약자)의 구조 공백**
- 시각장애인·이동약자는 스스로 SOS 발신이 어렵고, 주변 시민도 존재를 인지 못함
- 공공기관 복지 인프라와 시민 앱이 연결되지 않아 위기 상황 대응 공백 존재

**문제 ⑤: 생활 안전 시설물(소화기 등) 위치 정보 공백**
- 산·공원·건물 내 소화기 위치가 지도에 없어 초기 화재 대응 지연
- 네이버·카카오 지도는 상업 시설 중심으로 야외 헬스장·약수터·쉼터 등 비등록
- 지역민만 아는 생활 정보를 공유할 플랫폼 부재

## 2. 핵심 솔루션 (2~4페이지)

### 2.1 트로이 목마 전략: 일상 기능 → 안전망 전환

SpotVibe는 "지금 이 근처에 사람이 얼마나 있는지" 라는 일상적 궁금증을 해소하는 군집 밀도 지도를 핵심 일상 기능으로 제공합니다.

**[일상 기능으로 DAU 확보]**
- 연령대별 군집 밀도 시각화 (PRNG 기반 익명 분포)
- MBTI·활동 태그 오버레이 (개인화 탐색)
- AI 말풍선 인도 (능동 탐색 없이 앱이 먼저 말 걸기)

**[위기 시 자동 전환]**
- 반경 3km 내 앱 활성 사용자(DAU)에게 SOS 마커 실시간 노출
- 수신자가 "이미 앱을 켜고 있는 사람" → 유령 앱 문제 구조적 해결

### 2.2 SOS 4유형 분류 및 UI 설계

| 유형 | 코드 | 설명 | 공식 기관 선행 안내 |
|------|------|------|---------------------|
| 화재 | `fire` | 초기 화연 목격, 대피 필요 | 119 먼저 신고 안내 |
| 치안 | `public_safety` | 심야 위협, 범죄 연계 의심 | 112 먼저 신고 안내 |
| 조난 | `missing` | 고립, 이동 불가, 길 잃음 | 상황별 판단 |
| 구급 | `medical` | 의식 저하, 부상, 이동 불가 | 119 먼저 신고 안내 |

**Step 1**: 공식 응급기관 비대체 경고 + 119/112 단축 전화 → 동의 후 진행  
**Step 2**: 4유형 선택 + 메모 + 선택 사진  
**Step 3**: GPS 좌표 INSERT → Realtime 3km bbox 전파

### 2.3 취약계층 연계 및 생활 안전 시설물 레이어

**취약계층 근접 알림 시스템 (구청 연계)**
- 시각장애인·이동약자 등은 **구청(주민센터) 승인**을 통해 별도 등록 → 앱 활성화 시 상시 위치 공유
- 일반 사용자 반경 100m 내 취약계층 탐지 시 **진동 + 특별 알림 + 전용 마커** 표시
- 카테고리별 마커 구분: 시각장애인(지팡이), 이동약자(휠체어), 독거노인(집 아이콘) 등
- **"한 명이라도 더 도움받게"**: 앱을 켠 이웃 누구나 근처 취약계층 존재를 즉시 인지
- **MVP(구청 연계 전)**: 수신자 opt-in 기반 **짧은 문자형 이웃 도움 메시지·음성 안내**(`제품-기획/02_기능-현황-2026-05.md` §2.8)로 접근성·덕목 실증 서사 보강 가능

**생활 안전 시설물 시민 제보 맵**
- 소화기·화장실·야외 헬스장·공공 쓰레기통(분리수거함)·흡연실(흡연부스)·약수터 등을 GPS+사진으로 제보 → 카테고리별 지도 레이어
- **소화기 긴급 조회 버튼**: 탭 시 반경 3km 내 소화기 위치 즉시 표시 → 초기 화재 신속 대응
- SOS 화재 유형 발신과 소화기 조회 연계 안내
- 제보 경과 시간 비례 투명도 감쇠로 정보 신선도 시각화
- 마커 탭 시 **제보 실사진 + 제보 문구 + 거리**를 즉시 제공해 현장 신뢰도 강화
- 허위/오인 제보는 상이 신고자 누적 기준으로 관리하며 **5회 신고 시 자동 삭제/비노출** 처리

**동네 숨은 장소 카테고리 마커 (살아있는 지도)**
- 꽃·자연, 운동, 산책·풍경, 쉼터 등 카테고리별 제보 → 네이버 지도에 없는 정보 지도화
- **촬영 날짜 기반 계절 필터**: 철쭉·벚꽃 등 제철 명소 자동 추천 (이전 연도 동월 포함)
- 홈 화면 퀵 버튼: [SOS] [소화기] [꽃·자연] [운동] [공공쓰레기통] [장애인도움] 등

> **현장 증거 자료** (2026-05-08 안양 숲길 직접 촬영):
> 소화기 보관함, 야외 헬스장, 공원 분리수거함·쓰레기통, 수질부적합 약수터, 쉼터 도서관, 어린이 숲 체험원, 산불 주의 현수막, 식물 표본 구역 — 네이버 지도에 미등록된 실제 시설물 8종 이상 확인

### 2.4 신뢰성 및 허위 신고 방지 (다층 검증)

```
[Layer 1] 텍스트 LLM: SOS 메모 → allow / block (상업·성인·허위 등 차단)
[Layer 2] EXIF 무결성: 사진 촬영 시각·Software 태그 검사 (스크린샷·편집 차단)
[Layer 3] 비전 LLM: 화면 재촬영·허위 장면 판정 (verify-sos-photo Edge AI)
[Layer 4] 일일 한도: KST 기준 1일 1회 발신 제한 (DB 트리거)
[Layer 5] 집단 신고: 다수 신고 시 user_moderation 자동 연동
[Layer 6] 관리자 큐: held·신고 건 후행 검토 UI
[Layer 7] 시설물 마커 정화: 허위 신고 5회 임계 시 자동 비노출
```

## 3. 성장 모델 및 확산 전략 (2~3페이지)

### 3.1 사용자 심리 기반 바이럴 루프

| 단계 | 심리 | 앱 기제 |
|------|------|---------|
| 첫 접속 | "어디 사람 많지?" (외로움·FOMO) | 군집 밀도 지도 |
| 재방문 습관 | "오늘도 달라졌나?" | PRNG 드리프트 |
| SOS 목격 | "이게 뭐지?" (호기심) | 실시간 마커 |
| 사례 사진 확인 | "진짜 됐네, 나도?" | AI 검증 해결 사진 |
| 첫 참여 | "해봤다!" | 히어로 인증서·포인트 |
| SNS 공유 | "자랑하고 싶다" | 인스타 원터치 공유 |
| 주변 가입 | "나도 해볼게" | 바이럴 루프 완성 |

### 3.2 초기 히어로층 확보 전략

종교인, 자원봉사 경험자, 간호·의료 종사자 등 **이미 "도움"을 일상으로 하는 사용자**를 초기 사용자로 타깃합니다. 이들이 첫 사례를 쌓으면, 해결된 사례 사진이 새 SOS 마커에 연결되어 **일반 사용자의 자발적 참여를 유발**합니다.

### 3.3 콘텐츠 확산 (운영팀 주도)

- 주간 "히어로 인터뷰" 숏폼 (유튜브 쇼츠·인스타 릴스, 익명 시 3D 아바타 대체)
- 실제 해결 사례 → 언론 배포 → "앱이 아니라 문화" 포지셔닝

## 4. 기술 구현 현황 (3~5페이지)

### 4.1 현재 MVP 구현 완료 항목

| 분류 | 구현 내용 |
|------|-----------|
| 인증 | Supabase Auth + 카카오 OAuth |
| 지도 | Leaflet + Carto Dark Matter, GPS 내 위치, 군집 표시 |
| SOS 발신 | 4유형, GPS 좌표 INSERT, 15분 만료, KST 1일 1회 한도 |
| SOS 수신 | 반경 3km bbox Realtime 구독, 투명도 감쇠 마커 |
| AI 검증 | EXIF 무결성 검사 + 비전 LLM (verify-sos-photo) + 텍스트 LLM |
| 신고 체계 | sos_signal_abuse_reports, 집단 신고 → user_moderation 자동 연동 |
| 관리자 | SOS 모더레이션 페이지, held 큐, 이용자 제한 RPC |
| 개인정보 | 만료 후 신호 종료, 비식별 군집 표시, RLS 데이터 보호 |

### 4.2 기술 스택

```
Frontend  : React + TypeScript + Leaflet + Framer Motion
Backend   : Supabase (PostgreSQL, Realtime, RLS, Storage)
AI/Edge   : Supabase Edge Functions (Deno) + Groq (LLM, 비전)
Auth      : Supabase Auth + 카카오 OAuth
지도      : Leaflet + Carto (향후 Kakao/Naver Maps SDK 고도화 로드맵)
```

### 4.3 기술성숙도(TRL)

| 현재 | 목표 (과제 종료 시) |
|------|---------------------|
| TRL 4~5 (소규모 검증) | TRL 7 (실증 환경 실용화) |

## 5. 사업화 계획 및 수익 모델 (2페이지)

### 5.1 B2C — 무료 배포 (DAU·신뢰 확보)

일반 시민용 앱 무료 배포 → 대규모 트래픽 확보

### 5.2 B2G — SaaS 관제 대시보드 (구독 수익)

지자체 재난상황실·경찰·소방청에 **초정밀 LBS 재난 관제 대시보드** 연간 구독형 라이선스 공급

| 기능 | 설명 |
|------|------|
| 활성 SOS 집계 지도 | 지역·유형별 실시간 SOS 건수 |
| 군집 밀도 히트맵 | 비식별 인구 분포 데이터 (개인 식별 없음) |
| 감사 로그 | 신호 발신·해결·만료·신고 이력 |
| 알림 연동 | 임계치 이상 SOS 발생 시 담당자 자동 알림 |

### 5.3 장기 수익 구조

```
Phase 1 (0~1년): B2C 무료 앱 배포, 실증 데이터 축적
Phase 2 (1~2년): B2G 대시보드 파일럿 (지자체 1~3곳)
Phase 3 (2~3년): 전국 지자체 SaaS 확장, 이벤트 리스팅 광고
```

## 6. 기대 효과 및 성과 지표 (1~2페이지)

### 6.1 사회적 가치

- **골든타임 내 이웃 초기 대응 가능성 증대**: 출동 전 시민 대응 사례 수 측정
- **지자체 관제 비용 절감**: 하드웨어 없는 소프트웨어 관제 모델 실증
- **시민 안전 문화 확산**: SNS 바이럴·언론 보도를 통한 참여 문화 형성
- **취약계층 사회 안전망 강화**: 시각장애인·이동약자의 일상적 도움 접근성 향상 (구청 연계)
- **생활 안전 시설물 정보 공개화**: 소화기·약수터·야외 헬스장 등 비등록 시설 지도화 → 초기 화재 대응력 향상
- **지역 생활 정보 민주화**: 네이버·카카오 지도에 없는 비상업 생활 장소를 시민이 직접 지도화 → 지역 공동체 정보 격차 해소

### 6.2 정량 KPI (과제 제안)

| 지표 | 목표 (12개월 실증 기준) |

| 앱 DAU | 5,000명 이상 |
| SOS 발신 건수 | 월 50건 이상 |
| SOS 이웃 응답률 | 60% 이상 (반경 3km 내 수신자 존재 시) |
| 허위 신고 차단율 | 95% 이상 (AI 검증 기준) |

### 6.3 공공 안전 기여 서술

> "본 과제는 119/112 공식 응급기관을 대체하지 않습니다. 공식 출동 전 시민 초기 대응이 가능한 이웃을 연결함으로써, 골든타임 내 사망·피해 경감 가능성을 높이는 보조적 역할을 수행합니다. 모든 SOS 발신 시 공식 신고 병행을 UI에서 강제 안내합니다."
