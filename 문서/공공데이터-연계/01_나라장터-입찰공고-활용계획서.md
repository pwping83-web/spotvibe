> 위 외 오퍼레이션은 **이번 신청에서 제외**하며, 향후 필요 시 별도 신청합니다.

## 3. 화면·기능 구성 (텍스트 와이어프레임)

```
┌────────────────────────────────────────────────────────────┐
│  [이앤엑스 입찰공고 검색]                                  │
│  ┌──────────────┬──────────┬──────────┬──────────┐         │
│  │ 키워드       │ 업종     │ 발주기관 │ 마감일   │         │
│  └──────────────┴──────────┴──────────┴──────────┘         │
│  ── 최근 등록 공고 (목록 조회) ─────────────────────────── │
│  · 공고번호 / 공고명 / 발주처 / 예산 / 마감일 / 상세 보기  │
│  · 공고번호 / 공고명 / 발주처 / 예산 / 마감일 / 상세 보기  │
│  ── 마감 임박 알림 ──────────────────────────────────── │
│  · 관심 키워드 등록 시 마감 24/72시간 전 메일 알림         │
│  ── 출처 표기 ────────────────────────────────────────── │
│  본 정보는 「공공데이터포털 OpenAPI(조달청 나라장터)」를   │
│  통해 수집된 자료이며, 실제 입찰 진행은 나라장터 원문에서  │
│  확인하시기 바랍니다. → [나라장터 원문 바로가기]            │
└────────────────────────────────────────────────────────────┘
```

핵심 기능 4가지

1. **검색·필터** — 키워드/업종/발주처/예산/마감일 다중 조건
2. **북마크·알림** — 관심 공고 저장, 마감 임박 메일 알림
3. **변경 추적** — 공고 정정·연기 발생 시 이력 표시
4. **외부 링크 연계** — 상세 페이지 하단에 **나라장터 원문 바로가기** 버튼 고정

## 4. 데이터 처리·운영 흐름

```
[조달청 OpenAPI]
        │  (1) 정기 호출 (일 1~수 회, 페이지네이션 준수)
        ▼
[수집 서버 / 큐]
        │  (2) 정규화·중복 제거·해시 저장
        ▼
[검색 인덱스(자사 DB)]
        │  (3) 검색·필터 조회
        ▼
[웹 프론트엔드]
        │  (4) 사용자 화면 노출 + 출처 표기 + 원문 링크
        ▼
[사용자 / 메일 알림]
```

| 단계 | 통제 포인트 |
|---|---|
| (1) 호출 | **호출량 한도 준수** — 토큰 회전·증분 수집·캐시 |
| (2) 정규화 | 원문 필드명·값 보존, 변경 이력 별도 컬럼 |
| (3) 조회 | 자사 DB에 **공시 정보만** 저장(개인 위치·민감정보 미수집) |
| (4) 노출 | 출처 「공공데이터포털 OpenAPI(조달청 나라장터)」 명시, 원문 링크 의무 표기 |

## 5. 데이터 보호·법적 준수

| 항목 | 대응 |
|---|---|
| **개인정보** | 입찰공고에 포함될 수 있는 **담당자 이름·연락처는 화면 마스킹**(예: 김○○ / 02-***-1234). 회원 외 검색 결과에서는 마스킹 유지. |
| **민감정보** | 수집·저장하지 않음. |
| **위치정보** | 본 데이터셋은 **개인 위치정보를 포함하지 않으므로 「위치정보의 보호 및 이용 등에 관한 법률」상 위치기반서비스에 해당하지 않음.** 사용자 단말 위치 수집·저장 기능 없음. |
| **저작권·출처** | 공공저작물 자유이용허락(공공누리) 등 OpenAPI 상세 페이지 명시 조건 준수, 화면·검색결과·메일 알림에 **출처 표기**. |
| **서비스 약관** | 자사 이용약관·개인정보 처리방침에 본 OpenAPI 활용 사실 및 출처를 고지. |
| **변경 사항** | 공고 정정·연기 발생 시 **원문 우선** 안내, 자사 캐시는 보조 정보임을 명시. |

## 6. 호출·운영 한도 설계

- **호출 빈도**: 신규 공고 수집은 **하루 단위 정기 배치**(피크 시간대 회피).
- **장애 대응**: 응답 오류·타임아웃 시 **지수 백오프** 재시도, 일일 한도 80% 도달 시 알림.
- **트래픽 모니터링**: 호출 수·실패율·응답시간을 자체 대시보드에서 관리.
- **확장 시점**: 일평균 호출량이 한도의 70%를 지속 상회하면 **트래픽 증설 신청** 또는 **상세 호출을 사용자 클릭 시점으로 이연**하여 한도를 보존.

## 7. 기대효과

1. **기업 내부 효율** — 입찰 담당자가 일일 평균 30분 이상 단축, 마감 누락 사고 최소화.
2. **공공정보 활용 확산** — 분산된 공고 검색을 단일 검색·알림 인터페이스로 제공.
3. **공정한 시장 정보** — 중소기업의 정보 격차 해소, 마감 임박·정정 이력까지 함께 노출.
4. **데이터 무결성 유지** — 자사 캐시는 보조이며 **원문(나라장터)** 우선 원칙으로 정보 왜곡 방지.
