## 1. 서비스가 “사람만 군집”이 아닌 이유 (심사 담당자용 설명)

스팟바이브는 **실시간으로 사람이 몰리는 지점을 보여 주는 맛집 앱이나, 단순 군집 시각화만 하는 서비스가 아닙니다.**  
**지도·이벤트·AI 인사이트·동선 안내**를 묶어, 이용자가 **특정 지역에서 지금 어떤 분위기·행사·이동 참고 정보가 있는지**를 익명·통계 수준으로 확인할 수 있도록 설계된 **모바일 웹**입니다.

- **만남·소개팅·1:1 매칭 주선** 기능을 제공하지 않습니다.  
- **성인 전용 서비스**가 아닙니다.  
- **회원가입·로그인(카카오)·약관·개인정보 처리방침·회원 탈퇴** 등 일반 웹 서비스의 계정·고지 흐름을 갖추고 있습니다.

**[📷 여기에 캡처 붙이기: `/service` 서비스 안내 페이지 전체 — 위 설명이 보이도록 스크롤 포함]**

## 4. 작동 원리 한 블록 (텍스트만 넣어도 됨)

1. **위치:** 이용자가 동의한 범위에서 **내 위치** 또는 **탐색 지역**을 기준으로 지도·이벤트를 보여 줍니다.  
2. **타인 표시:** 정책에 따라 지도의 다른 이용자는 **근사·제한된 방식**으로만 표시할 수 있습니다(프라이버시 설계).  
3. **계정:** 카카오 OAuth 연동, **가입·로그아웃·탈퇴**가 가능합니다.  
4. **고지:** 약관·개인정보·사업자 정보를 **사이트 URL**로 제공합니다.

---

## 5. 제출 전 체크리스트

- [ ] **사이트 URL**이 주소창에 보이는 캡처가 최소 1장 이상 (`/service`, `/company` 등).  
- [ ] **이용 동선**이 순서대로 보이도록 배열 (가입 → 로그인 → 지도 → 마이 → 탈퇴 경로 → 고지).  
- [ ] **“단순 군집만”이 아님**을 설명하는 문장(위 §1·§4)이 첫 부분에 포함.  
- [ ] PDF/한 장 이미지 용량이 카카오 안내 범위 내인지 확인.

## 6. PDF 한 장으로 합치기 (이미 프로젝트에 있음)

로컬에서 웹 캡처를 `review-input-web` 폴더에 넣은 뒤:

```bash
npm run review:merge-full
```

→ `output/kakao-review-full-bundle.pdf` 생성. 앱 동선 이미지는 `review-input`의 `01.png`~`10.png`와 합쳐집니다.  
자세한 파일명 순서는 `scripts/review-images-to-pdf.mjs` 상단 주석 참고.
