## 2) 청구항 초안

출원 전략상 **방법 청구항**과 **시스템(장치) 청구항**을 분리해 두면, 침해 주장·회피 설계 대응 시 선택지가 넓어진다. 실제 번호·용어는 변리사가 선행조사 후 통합·재번호한다.

### 2.1 방법 청구항

#### 제1항 (독립항 — 방법)

지리 기반 사용자 생성 콘텐츠를 처리하는 컴퓨터 구현 방법으로서,

(a) 사용자 단말로부터 입력 텍스트를 수신하는 단계;  
(b) 상기 입력 텍스트를 텍스트 판정 모델에 입력하여 허용, 보류, 차단 중 하나로 분류하고, 보류인 경우 공개 전 검토 큐에 등록하는 단계;  
(c) 이미지가 첨부된 경우, 클라이언트에서 촬영 시각 메타데이터 및 캡처 의심 메타데이터를 검증하는 단계;  
(d) 상기 (c) 단계에서 통과한 이미지에 대해 비전 판정 모델로 남용 여부를 판정하는 단계;  
(e) 복수의 상이한 신고자 식별자에 기초한 신고 누적값이 임계치 이상이면 이용자 제한 상태를 갱신하는 단계; 및  
(f) 운영자 단말로부터 상기 보류 항목 또는 신고 항목에 대한 승인, 반려, 비공개 또는 제재 해제 중 적어도 하나의 조치를 수신하여 반영하는 단계;

를 포함하고, 하이퍼로컬 실시간 채널의 콘텐츠 무결성 또는 공익적 안전성 중 적어도 하나를 확보하는 것을 특징으로 하는 방법.

#### 제2항 (종속)

제1항에 있어서, 상기 (b) 단계에서 보류로 분류된 경우 공개 피드에 상기 콘텐츠를 노출하지 않고, 관리자 검토 완료 후에만 공개 상태로 전이시키는 것을 특징으로 하는 방법.

#### 제3항 (종속)

제1항에 있어서, 상기 (c) 단계는 촬영 시각이 사전 정의된 시간 창 내에 있는지 여부 및 스크린샷 생성 소프트웨어 태그 포함 여부를 확인하는 것을 특징으로 하는 방법.

#### 제4항 (종속)

제1항에 있어서, 상기 (e) 단계는 상기 신고 누적값을 서로 다른 신고자 식별자의 수로 산정하고, 상기 임계치 도달 시 기존 제재 상태가 미리 정한 우선순위에서 더 높은 단계이면 해당 상태를 유지하는 것을 특징으로 하는 방법.

#### 제5항 (종속)

제1항에 있어서, 지도 인터페이스에서 사용자 성향 속성 집합을 복수의 지리 오프셋 및 반경에 매핑하여 복수의 오버레이를 동시에 렌더링하는 단계를 더 포함하는 것을 특징으로 하는 방법.

#### 제6항 (종속)

제1항에 있어서, 현재 시각에 대응하는 시간 버킷 값을 난수 시드로 사용하여 클라이언트 간 동기화 가능한 군중 분포 좌표를 생성하고, 연령대별로 구분된 마커로 지도에 표시하는 단계를 더 포함하는 것을 특징으로 하는 방법.

### 2.2 시스템(장치) 청구항

#### 제1항 (독립항 — 시스템)

지리 기반 사용자 생성 콘텐츠를 처리하는 정보처리시스템으로서,

텍스트 수신부;  
상기 입력 텍스트를 허용, 보류, 차단으로 분류하고 보류 시 검토 큐에 등록하는 텍스트 판정부;  
이미지 첨부 시 클라이언트 측에서 촬영 시각 메타데이터 및 캡처 의심 메타데이터를 검증하는 메타데이터 검증부;  
상기 메타데이터 검증부의 통과 이미지에 대해 남용 여부를 판정하는 비전 판정부;  
상이한 신고자 식별자에 기초한 신고 누적값이 임계치 이상일 때 이용자 제한 상태를 갱신하는 신고 집계부; 및  
운영자 단말로부터 보류 또는 신고에 대한 승인, 반려, 비공개 또는 제재 해제 중 적어도 하나의 조치를 수신하여 반영하는 운영 조치부;

를 포함하고, 하이퍼로컬 실시간 채널의 콘텐츠 무결성 또는 공익적 안전성 중 적어도 하나를 확보하도록 구성된 것을 특징으로 하는 정보처리시스템.

#### 제2항 (종속)

제1항에 있어서, 상기 텍스트 판정부는 보류 분류 시 공개 피드 노출을 차단하고 관리자 검토 완료 후 공개 상태로 전이시키도록 구성된 것을 특징으로 하는 정보처리시스템.

#### 제3항 (종속)

제1항에 있어서, 상기 메타데이터 검증부는 촬영 시각의 시간 창 적합 여부 및 스크린샷 생성 소프트웨어 태그 포함 여부를 검사하도록 구성된 것을 특징으로 하는 정보처리시스템.

#### 제4항 (종속)

제1항에 있어서, 상기 신고 집계부는 서로 다른 신고자 식별자 수로 누적값을 산정하고, 임계치 도달 시 기존 제재 상태의 우선순위가 더 높으면 해당 상태를 유지하도록 구성된 것을 특징으로 하는 정보처리시스템.

#### 제5항 (종속)

제1항에 있어서, 사용자 성향 속성 집합을 복수의 지리 오프셋 및 반경에 매핑하여 지도에 복수 오버레이를 동시 렌더링하는 지도 표시부를 더 포함하는 것을 특징으로 하는 정보처리시스템.

#### 제6항 (종속)

제1항에 있어서, 현재 시각에 대응하는 시간 버킷 값을 난수 시드로 사용하여 군중 분포 좌표를 생성하고 연령대별 마커로 지도에 표시하는 군중 시뮬레이션부를 더 포함하는 것을 특징으로 하는 정보처리시스템.

#### 제7항 (BM 시스템 독립항 — 일상 DAU 기반 시민 구조자 전환)

위치 기반 군집 밀도 정보를 제공하는 플랫폼에서 이웃 도움 신호를 운영하는 정보처리시스템으로서,

일상 군집 탐색 기능으로 활성 사용자(DAU) 풀을 상시 확보하는 일상 기능부;
상기 DAU 중 도움 신호 발신자의 위치·유형을 수신하여 반경 내 활성 사용자에게 실시간 전파하는 신호 전파부;
AI 검증된 과거 해결 사례 이미지를 도움 신호 수신 화면에 연결하여 사회적 증거를 제공하는 증거 노출부; 및
참여 완료 시 디지털 인증서를 발급하고 소셜 미디어 공유 링크를 생성하는 바이럴 유도부;

를 포함하고, 일상 플랫폼 DAU를 비상 상황 이웃 구조자 풀로 전환하도록 구성된 것을 특징으로 하는 정보처리시스템.

### 5.1 방법 청구항 — 취약계층 근접 알림 (제8항)

#### 제8항 (독립항)

위치 기반 서비스 플랫폼에서 취약계층 안전을 지원하는 컴퓨터 구현 방법으로서,

(a) 공공기관 담당자의 승인을 조건으로 취약계층 사용자를 일반 사용자와 분리된 테이블에 카테고리 및 승인 기관 정보와 함께 등록하는 단계;
(b) 등록된 취약계층 사용자의 단말에서 고정밀 위치 서비스(maximumAge:0, enableHighAccuracy:true)로 위치를 주기적으로 갱신하여 서버에 저장하는 단계;
(c) 일반 사용자 단말에서 앱이 활성화된 상태로 현재 위치를 조회하고, 설정 반경 이내에 등록 취약계층 위치가 존재하는지 판정하는 단계;
(d) 상기 (c) 단계에서 반경 내 취약계층이 탐지되면, 카테고리별 전용 아이콘 및 색상 코드 마커를 지도에 표시하고, 진동 패턴 및 푸시 알림을 복합 발동하는 단계; 및
(e) 취약계층 위치 정보를 일반 사용자에게 제공 시 좌표만 노출하고 개인 식별 정보는 행 수준 보안(RLS) 정책으로 차단하는 단계;

를 포함하는 것을 특징으로 하는 공공기관 연계 취약계층 근접 안전 알림 방법.

#### 제8-1항 (종속)

제8항에 있어서, 상기 취약계층 카테고리는 시각장애인, 청각장애인, 이동약자, 독거노인, 미아·실종 중 적어도 하나를 포함하고, 카테고리마다 상이한 반경 임계값과 마커 디자인을 적용하는 것을 특징으로 하는 방법.

---

### 5.2 방법 청구항 — 생활 안전 시설물 제보 레이어 (제9항)

#### 제9항 (독립항)

위치 기반 서비스 플랫폼에서 생활 안전 시설물을 관리하는 컴퓨터 구현 방법으로서,

(a) 사용자 단말에서 GPS 활성 상태로 촬영된 사진의 EXIF 위치 메타데이터를 추출하여 카테고리(소화기, 화장실, 야외 헬스장, 공공 쓰레기통·분리수거함, 약수터 등)와 함께 서버에 저장하는 단계;
(b) AI 텍스트 판정 및 비전 판정을 통해 제보 내용의 적합성을 검증하는 단계;
(c) 제보 저장 시각과 현재 시각의 차이에 비례하여 지도 마커의 투명도를 감쇠시키는 단계;
(d) 소화기 카테고리에 대해 홈 화면 긴급 조회 버튼을 제공하여 탭 시 설정 반경 내 소화기 위치를 거리순으로 표시하는 단계; 및
(e) 사용자가 "현재도 있음" 버튼을 탭하면 해당 제보의 freshness 갱신 시각을 업데이트하는 단계;

를 포함하는 것을 특징으로 하는 시민 제보 기반 생활 안전 시설물 위치 레이어 운영 방법.

#### 제9-1항 (종속)

제9항에 있어서, 화재 유형 SOS 신호 발신 단계 내에서 소화기 긴급 조회 연계 안내를 제공함으로써, 공식 소방 대응 전 시민의 초기 화재 대응 역량을 지원하는 것을 특징으로 하는 방법.

#### 제9-2항 (종속)

제9항에 있어서, 지도상의 시설물 마커를 선택하면 해당 시설물의 제보 사진, 제보 시각, 제보자 식별자 요약 정보 및 설명 문구를 포함하는 상세 시트를 표시하는 단계를 더 포함하는 것을 특징으로 하는 방법.

#### 제9-3항 (종속)

제9항 또는 제9-2항에 있어서, 동일 시설물 제보에 대한 상이한 신고자 식별자 기반 허위 신고 누적값이 사전 정의 임계치(예: 5회)에 도달하면 해당 시설물 마커를 자동 비노출 또는 삭제 상태로 전이하는 단계를 더 포함하는 것을 특징으로 하는 방법.

---

### 5.3 방법 청구항 — 카테고리·계절 기반 장소 지도화 (제10항)

#### 제10항 (독립항)

위치 기반 서비스 플랫폼에서 비상업적 생활 장소를 지도화하는 컴퓨터 구현 방법으로서,

(a) GPS 활성 상태에서 촬영된 사진의 위치 및 날짜 메타데이터와 함께 사용자가 선택한 카테고리 및 서브카테고리를 서버에 저장하는 단계;
(b) 현재 월을 기준으로 설정 범위의 월 필터를 적용하여 해당 계절에 적합한 제보를 우선 표시하고, 이전 연도 동월 제보를 포함하여 계절성 장소를 반복 추천하는 단계;
(c) 지도 줌 레벨에 따라 줌아웃 시 카테고리별 클러스터 집계 원을, 줌인 시 개별 마커와 썸네일 미리보기를 전환 표시하는 단계; 및
(d) 홈 화면 카테고리 퀵 버튼 탭 시 해당 카테고리 마커 레이어만 활성화하고, 비활성 카테고리는 지도에서 숨겨 지도 복잡도를 제어하는 단계;

를 포함하는 것을 특징으로 하는 시민 제보 기반 지역 장소 카테고리 지도화 방법.

### 5.4 변리사 전달용 3종 요약

**① 취약계층 근접 알림 시스템**
공공기관 승인 기반 취약계층 위치 상시 공유, 반경 100m 내 일반 사용자에게 카테고리별 전용 마커 + 복합 알림 발동. 개인정보는 RLS로 좌표만 노출하는 공익 특화 위치 서비스.

**② 생활 안전 시설물 제보 레이어**
시민이 GPS+사진으로 소화기·화장실·야외 운동시설·공공 쓰레기통(분리수거함)을 제보하면 카테고리별 지도 레이어를 구성하고, 마커 클릭 시 제보 사진·설명·거리 정보를 즉시 제공한다. 허위 신고 누적 5회 임계 도달 시 자동 삭제/비노출 처리로 데이터 신뢰도를 유지한다.

**③ 카테고리·계절 기반 장소 지도화**
비상업적 생활 장소(꽃길·야외 헬스장·쉼터)를 시민 제보로 카테고리화 + 현재 월 기반 계절 필터 + 이전 연도 동월 포함 추천. 줌 레벨 연동 클러스터/개별 마커 전환.
