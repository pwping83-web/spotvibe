# 서비스 끊김·전체 장애 — 즉시 체크리스트

“앱이 아예 안 된다”일 때 **위에서 아래로** 진행합니다.

---

## 1단계: 범위 확인 (1분)

- [ ] **본인만** 안 되는지, **팀 전체·외부 사용자**도 같은지 (다른 네트워크·LTE로 확인).
- [ ] 브라우저 **시크릿 창**에서 동일 증상인지 (캐시·확장 프로그램 배제).

---

## 2단계: 정적 사이트(프론트) 살아 있는지

배포 URL을 연다.

- [ ] **메인 HTML**이 로드되는지 (완전 백지면 호스팅/CDN/DNS).  
- [ ] 호스팅 **상태 페이지**(Netlify/Vercel 등)에 인시던트 없는지.  
- [ ] 최근 **DNS·도메인 만료** 변경이 없었는지.

**실행 예 (로컬에서 DNS만 확인)**

```powershell
nslookup 당신의-프로덕션-도메인
```

---

## 3단계: Supabase 상태

- [ ] [Supabase 상태 페이지](https://status.supabase.com/) (공식)에서 리전 이슈 없는지.  
- [ ] 프로젝트 **Dashboard** 로그인 가능한지.  
- [ ] **Database** 메뉴에서 인스턴스 일시 정지·결제 이슈 메시지 없는지.

**자주 보는 증상**

| 증상 | 의심 지점 |
|------|-----------|
| 로그인만 안 됨 | Auth 설정, JWT 만료, 카카오 콘솔 Redirect URI |
| 데이터만 안 됨 | RLS 오류, 마이그레이션 미적용 환경 |
| 전부 타임아웃 | 리전 장애, 네트워크, 잘못된 API URL |

---

## 4단계: 클라이언트 환경 변수

- [ ] 프로덕션 빌드에 **`VITE_*` Supabase URL·anon 키**가 올바른 프로젝트를 가리키는지.  
- [ ] 실수로 **개발용 프로젝트**를 가리키지 않았는지 (배포 직후 장애에 흔함).
