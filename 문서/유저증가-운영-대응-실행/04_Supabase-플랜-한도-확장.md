# Supabase 플랜·한도 — 확인과 확장 실행

수치·메뉴명은 **Supabase 제품 업데이트**로 바뀔 수 있으므로, 아래는 **무엇을 보면 되는지** 위주입니다. 최종 한도는 **대시보드·공식 가격/문서**를 따릅니다.

---

## 1. 대시보드에서 볼 항목 (순서)

- [ ] **Billing / Usage** (또는 동일 목적 메뉴): Database, Storage, Egress, MAU 등.  
- [ ] **Database**: CPU·디스크·연결 수 경고.  
- [ ] **Auth**: 월간 활성 사용자(MAU) 관련 한도.  
- [ ] **Realtime**: 동시 채널·메시지 한도(플랜별).

---

## 2. 증상과 대응 (요약)

| 증상 | 흔한 원인 | 실행 |
|------|-----------|------|
| 429 Too Many Requests | API 레이트 / 플랜 한도 | 사용량 그래프 확인 → 플랜 상향 또는 쿼리·구독 최적화 |
| DB 타임아웃 | 느린 쿼리·인덱스 부재·동시 접속 | 쿼리 최적화, 인덱스, 읽기 복제(엔터프라이즈 등) |
| Storage 업로드 실패 | 용량 한도 | `03_스토리지-용량-부족-실행.md` + 플랜 |
| Edge Function 오류 | 콜드 스타트·타임아웃·외부 API | 로그 확인, 타임아웃·재시도 설정 |

---

## 3. 확장 결정 시 체크

- [ ] **스테이징 프로젝트**에서 플랜/설정 변경 먼저 검증할지.  
- [ ] 비용 상한(월 예산) 합의.  
- [ ] 업그레이드 후 **스모크 테스트**: 로그인, 지도, 제보 업로드, SOS 흐름.

---

## 4. 장기 (유저가 크게 늘 때)

- 읽기 많은 데이터는 **캐시·CDN·집계 테이블**로 DB 부하 분산 검토.  
- **다리전(리전)** 이전은 까다로움 → 초기에 리전 선택을 신중히.
