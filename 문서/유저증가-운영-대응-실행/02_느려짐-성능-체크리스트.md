---

## A. 백엔드(Supabase) 쪽

- [ ] Dashboard → **Reports / Query Performance**(메뉴명은 시점별로 상이)에서 **느린 쿼리·CPU** 확인.  
- [ ] **Realtime** 구독 수가 비정상적으로 많지 않은지 (지도에서 구독 해제 누락 시 증가).  
- [ ] 동시 접속 급증 시 **커넥션 풀·플랜 한도** → `04_Supabase-플랜-한도-확장.md`.

**실행(개발자)**

- [ ] 문제 재현 구간에서 **Network 탭**으로 어떤 요청이 긴지 확인 (PostgREST vs Storage vs Edge).

---

## B. 지도·마커(프론트)

SpotVibe는 **Leaflet + 다수 마커**를 쓰므로, 유저·제보가 늘면 **클라이언트 렌더 부하**가 커질 수 있습니다.

- [ ] 한 화면에 그리는 **마커 수 상한**·클러스터링·뷰포트 밖 제거 로직이 있는지 코드 확인.  
- [ ] **불필요한 setState / 전체 리렌더** 루프가 없는지 (개발자 도구 Performance).

---
