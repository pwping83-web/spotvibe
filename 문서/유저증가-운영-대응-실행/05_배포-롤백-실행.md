# 배포·롤백 — 실행 가이드 (Vite 정적 호스팅)

SpotVibe 프론트는 **Vite** 빌드 결과를 정적 호스팅에 올리는 형태입니다 (`문서/제품-기획/04_기술-구조.md`).

---

## 1. 배포 전 (5분)

```bash
npm run build
```

- [ ] 빌드 **에러·경고** 없음.  
- [ ] `npm run dev`로 **로그인·지도·제보·SOS** 스모크 테스트 (배포 브랜치 기준).

---

## 2. 배포 직후 장애 시 — 롤백

호스팅마다 UI가 다릅니다. **공통 개념**:

- [ ] 호스팅 대시보드 → **Deployments / 릴리스 목록**에서 **이전 성공 배포**로 “Promote / Rollback”.  
- [ ] **DNS 또는 프리뷰 URL**로 이전 버전이 정상인지 확인한 뒤 프로덕션 전환.

**Git 기준 롤백(소스 되돌리기)**

```bash
git revert HEAD
# 또는 특정 커밋으로 되돌린 뒌 새 PR/배포
```

팀 규칙에 따라 **`git reset --hard` + force push**는 피하고, **revert**를 권장합니다.

- [ ] Supabase **프로젝트 ref**가 프로덕션과 일치하는지.
