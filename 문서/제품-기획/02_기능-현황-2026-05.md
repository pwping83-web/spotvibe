## 2. 기능 영역별 정리 (현재 구현)

### 2.1 인증·계정

- **Supabase Auth** + **카카오 OAuth** (`App.tsx` 라우팅, 세션·`authReady`).
- **모더레이션 강제**: `user_moderation` 상태에 따라 로그아웃·안내 (`useModerationEnforcement`).
- **관리자**: 특정 이메일 기준 관리자 라우트·지도 테스트·SOS 일일 한도 예외 등(코드 내 정책).

### 2.2 지도·탐색 (MapArea 등)

- **배경 지도**: `VITE_KAKAO_MAP_JS_KEY` 설정 시 **카카오맵(JavaScript API)** 래스터 + **Leaflet**은 타일 없이 마커·원·폴리라인만(동기화). 키 미설정 시 **Carto Dark Matter**(OSM 기반) 타일.
- **내 위치** / **탐색 중심** 모드.
- 연령대 **PRNG 점 분포**(1분 시드 드리프트), **MBTI·활동 태그** 오버레이, **심장 박동** 스케일 애니메이션.
- **AI 추천 패널**, **AI 말풍선 인도**, 클러스터 탭 → **BottomSheet**(이동 프로필별 문구·네이버 링크).
- **이벤트 핀**·지역 프리셋 등(일부 데이터는 시연/더미 성격 유지 가능).

### 2.3 현장 제보(Spot Report)

- **업로드**: 위치(신선도 옵션)·사진·(선택) 장소명·설명, **얼굴 모자이크** 후 `spot-photos` 업로드.
- **텍스트 AI**: `moderate-user-content` → `allow` / **`held`** / `block`. `held`는 공개 전 검토, `block`은 차단.
- **검증·포인트**: `pending` → `verified` 등, 트리거로 **기여 포인트** 가산 경로.
- **신고·숨김**: `flag_spot_report` 등, 다인 신고 시 **`hidden`** 등 자동 처리.
- **실시간 주변 사진**: 반경·시간 창 조회, 좋아요, **유형별 신고**(`submit_content_abuse_report`), 관리자 삭제 등.

### 2.4 SOS(이웃 도움 신호)

- **4유형**: 화재·치안·조난·구급 (`src/types/sos.ts`).
- **발신 시트(`SosSignalSheet`)**: 1단계에서 공식 응급 안내 문구 + **119/112 전화**(연결 전 `confirm`) + 위치 수신 후에만 활성화되는 **「확인했습니다」**로 유형 선택(2단계) 진입.
- **일일 한도**: KST 기준 1일 1회(DB 트리거), **관리자 예외** 마이그레이션.
- **메모**: `moderate-user-content`(`sos_note`) — 차단 시 발신 불가.
- **사진(선택)**:  
  - 클라 **EXIF 촬영 시각·스크린샷 Software** 검사(`sosPhotoIntegrity.ts`).  
  - 통과 후 압축 JPEG → **`verify-sos-photo`**(비전).  
  - 스토리지 `sos-photos`.
- **실시간**: 반경 3km bbox, Realtime, 만료 15분, 마커 **투명도 감쇠**, 활성 시 **진동·알림·전체화면 오버레이**(지도 거의 투명).
- **피어 시트**: 타인 신호 확인·응답·**유형별 신고**(범죄 연계 등).
- **부정 이용**: `sos_signal_abuse_reports` 등 + **서로 다른 신고자 수** 임계 시 `user_moderation` 연동(마이그레이션 정의).

### 2.5 매칭·포인트·마이

- **바이브 부스트(peek 세션)**: 포인트 차감·48h·필터 반영 RPC (`peek_match_session` 계열).
- **근접 매칭** 후크, `last_seen_at` / `updated_at` 신선도.
- **마이**: 프로필·알림 요일·운영 문의·**관리자 모더레이션** 진입 등.

### 2.6 관리자·운영

- **지도 테스트 모드**(마이·관리자): 가상 인구·데모 시설과 함께 **근처 화재 SOS 시뮬 마커**(클라 전용·DB 없음, 탭 시 피어 시트 미연동·토스트 안내).
- **SOS / 현장 제보 모더레이션 페이지**: AI 보류(`held`) 목록, `content_abuse_reports`, RPC로 승인·반려·hidden.
- **Edge**: `moderate-user-content`, `verify-sos-photo`(배포·Secrets 필요). 기타 `admin-delete-spot-report` 등.
- **데이터 수집 패널** (`AdminSosModerationPage` "데이터 수집" 탭): 수집 이력 조회 + `sync-public-notices` 수동 트리거 (전체·재난문자만·지역행사만).

### 2.7 기타 UI·라우팅

- 탭: 지도 / 이벤트 / 알림 / 마이, **SOS 후기** 페이지, **개인정보·약관** 등 정적 라우트.
- **로드맵 문서**의 “서버 히트맵 WebSocket” 등은 **미구현 아이디어**로 남을 수 있음 → §3 참고.

### 2.8 취약계층 위치 공유·근접 알림 시스템 (✅ 이웃 도움 메시지·음성 MVP · 구청 연계·실계정 VI는 로드맵)

> 아이디어 원안: 2026-05-08 현장 탐색 중 기획. 시각장애인·이동약자 등 취약계층의 안전을 위해 이웃의 자발적 도움을 즉각 연결하는 시스템.

#### 개요

- **별도 가입 경로**: 시각장애인 등 대상자는 **구청(주민센터) 승인**을 통한 별도 등록. 일반 앱 가입과 분리된 공공기관 연계 인증 흐름.
- **상시 위치 공유**: 등록 승인 후 앱이 활성화된 상태에서 **GPS 위치를 상시 공유**. 일반 사용자 대비 위치 갱신 주기를 더 높은 정밀도로 유지.
- **근접 알림**: 일반 사용자가 앱을 켜고 있을 때 **반경 100m 이내** 등록 취약계층이 있으면 **진동 + 특별 알림** 발동.
- **특별 마커**: 지도에 취약계층 위치를 **일반 마커와 구분되는 전용 마커**(지팡이/도움 아이콘, 강조 색상)로 표시. 누구나 즉시 식별 가능.
- **홈 화면 안내 버튼**: 홈에 전용 버튼 → 탭 시 제도 안내, 가입 절차, 도움 방법 설명 화면 표시.
- **정밀 GPS 우선**: 취약계층 단말에서는 maximumAge:0, enableHighAccuracy:true 강제 적용. 위치 갱신 빈도 높게 설정.

#### 특수케이스 확장

| 케이스 | 마커 | 반경 알림 |
|--------|------|-----------|
| 시각장애인 (구청 승인) | 지팡이 아이콘 + 파란 테두리 | 100m |
| 청각장애인 (구청 승인) | 귀 아이콘 + 초록 테두리 | 100m |
| 이동약자·휠체어 | 휠체어 아이콘 + 노란 테두리 | 100m |
| 미아·실종 신고 연계 | 경고 아이콘 + 주황 테두리 | 300m |
| 독거노인·복지 대상 (구청 연계) | 집 아이콘 + 보라 테두리 | 100m |

#### 구현 고려사항

- **개인정보·위치정보**: 취약계층 위치는 최고 등급 개인정보. 공공기관 인증 연계, RLS 별도 테이블, 기관 승인 계정만 등록 가능.
- **배터리**: 고정밀 상시 추적은 배터리 소모 큼. PWA + 백그라운드 주기 업데이트 혼용 또는 전용 단말 배포 방안 검토.
- **오남용 방지**: 구청 승인 없는 임의 등록 불가. 관리자 승인 RPC + 공공기관 인증 연동.

#### MVP 구현 (2026-05-13) — 짧은 이웃 도움 메시지 + 음성 안내

#### facilityMarkers 통합 state (MapArea.tsx)

소화기·화장실·야외운동·**공공 쓰레기통(`trash`)**·**흡연실(`smoking`)** 마커를 단일 `facilityMarkers` state로 통합 관리.
- **type**: `'fire' | 'toilet' | 'gym' | 'trash' | 'smoking'`
- **demoAroundCenter**: 데모 시설(화장실·쓰레기통·흡연·야외운동)만 `true` — 지도 핀은 탐색/내 위치 중심 주변에 배치되어, 지역을 바꿔도 500m 원 안에 보임(저장 좌표와 다를 수 있음)
- **photoUrl**: 제보자가 찍어 올린 실제 사진 URL (현재 데모 사진 사용, 실서비스 시 Supabase Storage 연동 필요)
- **reportCount**: 허위 신고 누적 수 — **5회 도달 시 자동 삭제**
- **gone**: 없음/폐쇄 신고 처리 상태

#### 마커 클릭 → 제보 사진 시트 (`MarkerPhotoSheet`)

마커 탭 시 해당 제보의 실제 사진·설명·제보자 정보를 표시.
- 제보자 닉네임 · 날짜 · 거리
- 제보 사진 (실사용자 업로드 기반 — 현재는 데모 사진 시뮬레이션)
- 신고 버튼: 누적 5회 → 자동 삭제 + 진행 바(3회 이상 빨간색 경고)

> **실사진 적용 계획**: 실서비스에서는 `ToiletSheet`·`FireExtinguisherSheet`의 사진 업로드 → Supabase Storage 저장 → `facilityMarkers`에 photoUrl로 연결. 현재 UI/UX 구조는 실제 제보 사진 기반으로 동작하도록 설계 완료.

## 3-b. SOS 성장 루프 설계 — "외로움 DAU → 구조자 전환" 전략

> 이 섹션은 SOS 기능이 단독으로 기획된 것이 아니라, **일상 DAU 확보 루프와 유기적으로 연결된 성장 전략**임을 기록합니다.

### 핵심 명제

평상시 사용할 이유가 없는 SOS 전용 앱은 위기 시 주변에 수신자(구조자)가 0명인 구조적 실패를 낳습니다.
SpotVibe는 이 문제를 **"일상 기능으로 상시 대기층을 만든다"** 는 방식으로 해결합니다.

```
[상시 DAU 확보]
  군집 밀도 지도 → 외로움·FOMO → 습관적 재방문
  → SOS 발생 시 반경 3km 내 "지금 앱을 켜고 있는 이웃" 존재 확률 대폭 상승

[수동 관찰자 → 능동 구조자 전환]
  SOS 마커 호기심 탭
  → AI 검증 해결 사례 사진 노출 (social proof)
  → "나도 할 수 있다" 자기효능감 유발
  → 첫 구조 참여

[바이럴 루프]
  히어로 인증서 → SNS 공유 → 신규 가입 → DAU 증가 → SOS 해결률 상승 → 루프
```

### SOS 사례 사진의 역할 (현재 구현 기반)

SOS 피어 시트(`SosPeerSignalSheet.tsx`)에서 첨부 사진을 표시하는 기능은 이미 구현되어 있습니다.
이 사진이 **과거 해결 사례**와 연결될 때 "사회적 증거"로 기능하는 것이 핵심입니다.

| 구현 상태 | 내용 |
|-----------|------|
| ✅ 완료 | SOS 마커 탭 → 피어 시트 열림, 첨부 사진 표시 |
| ✅ 완료 | EXIF + 비전 LLM 검증 (허위·조작 차단) |
| ✅ 완료 | 4유형 분류(화재·치안·조난·구급) + 안내 메시지 |
| 🔜 로드맵 | 해결 완료 사례를 다음 SOS 피어 시트에서 "이전 사례" 형태로 노출 |
| 🔜 로드맵 | 히어로 인증서 발급 + SNS 원터치 공유 |
| 🔜 로드맵 | 누적 구조 등급 (브론즈→실버→골드 가디언) |

### 초기 히어로층 타깃

종교인, 자원봉사 경험자, 간호·의료 종사자 등 **이미 "도움"을 일상으로 하는 사용자**가 초기 구조 사례를 쌓습니다.
이 사례들이 지도에 누적되면, 일반 사용자에게도 "이 앱은 실제로 작동한다"는 신뢰가 형성됩니다.
