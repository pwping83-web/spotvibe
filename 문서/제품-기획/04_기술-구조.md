# 스팟바이브 — 구동 방식 및 기술 구조

## 실행 환경

- **Node.js** 권장: LTS 계열 (프로젝트는 Vite 6 기준)
- **패키지 매니저**: `npm` 또는 `pnpm` (저장소에 `pnpm` 오버라이드 설정 있음)

## 구동 방법

프로젝트 루트(`SpotVibe`)에서:

```bash
npm install
npm run dev
```

개발 서버가 뜨면 **기본 브라우저가 자동으로 열립니다**(Vite `server.open: true`). 주소는 터미널의 `Local:` 안내를 따르면 됩니다(보통 `http://localhost:5173`; 포트가 잡혀 있으면 다음 번호).

**이 환경에서 쓰는 로컬 주소(저장):** `http://localhost:5174`

**기본 브라우저와 Cursor 안(Simple Browser)을 같이 열 때** (`npm run dev` 실행 중):

```bash
npm run open:app
```

5173이 아닌 포트로 떠 있으면(예: 5174):

```bash
npm run open:app -- 5174
```

`scripts/open-app.ps1`이 `Start-Process`로 시스템 브라우저를 열고, `cursor` CLI로 `vscode://vscode.simple-browser/show?...` 를 호출합니다. 터미널에서 `cursor`를 찾을 수 없다면 Cursor에서 **Shell Command: Install 'cursor' command in PATH**(또는 동일 목적 메뉴)로 CLI를 설치하세요.

프로덕션 빌드:

```bash
npm run build
```

빌드 결과물은 `dist/`에 생성됩니다. 정적 호스팅(Netlify, Vercel, S3+CloudFront 등)에 그대로 올릴 수 있는 형태입니다.

## 앱이 동작하는 순서

1. **`main.tsx`**  
   `leaflet.css`와 전역 스타일을 불러온 뒤, 루트에 `<App />`을 마운트합니다.

2. **`App.tsx`**  
   - `AuthProvider`로 로그인 여부를 메모리 상태로 관리합니다.  
   - `/login` 미로그인 시 로그인 페이지, `/` 로그인 시 메인 셸(`SpotVibeMain`)로 분기합니다.  
   - 최대 너비 약 430px의 “폰 프레임” 레이아웃으로 모바일 UX를 가정합니다.

3. **메인 탭**  
   `NavigationBar`로 `map` / `events` / `notifications` / `my` 전환.  
   `motion`으로 탭 전환 시 페이드·슬라이드 전환.

4. **지도 탭 (`MapArea.tsx`)**
   - `MapContainer` + `TileLayer` + Carto Dark Matter.
   - 연령대별 색의 `CircleMarker`(투명 군집), 이벤트 `Marker`, 클러스터 `Circle`.
   - **심장 박동**: MBTI·성별·활동 원이 `hbTick`(95ms) 기반 `sin` 파형으로 맥박 애니메이션.
   - **내 위치**: GPS 켜면 파란 점 + 「내 위치」라벨, 없으면 탐색 중심에 「탐색 중심」라벨.
   - **AI 인도 말풍선**: 30~58초 랜덤 간격으로 가끔만 「여기 어때? / 여기는?」 말풍선 + 화살표·점선 경로 표시.
   - **AI 추천 버튼**: 우하단 버튼 탭 시 마이 프로필(MBTI·활동·성별)로 스코어링한 후보 순서대로 카드 표시, 30초 카운트다운 바, 「다음」으로 순환.
   - **탐색 중심**: 상태로 중심 좌표를 바꾸면 점·클러스터·이벤트 핀·AI 추천 앵커가 같이 이동.

5. **클러스터 탭 → BottomSheet**
   `onClusterClick`으로 `showAIInsight`가 켜지고, 지도 탭일 때만 `BottomSheet`가 열립니다.
   시트 내용은 `mobilityProfile`에 따라 경로 안내 문구가 달라지고, 네이버 지도 검색 URL 헬퍼로 외부 링크를 제공합니다.

6. **마이 (`MyPage.tsx`)**
   MBTI·혈액형·활동 태그·성별 인파·이동 프로필·선호 시간 설정이 지도 오버레이와 AI 추천 스코어에 연결됩니다.
   하단 **운영·문의** 섹션(개선 제안·광고·행사·기타)에서 메일 앱으로 접수할 수 있습니다.

7. **헤더 (`App.tsx`)**
   `스팟바이브 ←── 실시간 시계 ──→ 현재 탭` 한 줄 바. 시계는 `useLiveClock` 훅으로 10초마다 갱신.

## 데이터·백엔드 현황

- **Supabase(PostgreSQL + Auth + Storage + Realtime)**  
  프로필·위치·현장 제보(`spot_reports`)·SOS(`sos_signals`)·신고·모더레이션·포인트 등은 **마이그레이션 기준 스키마**로 연동 가능한 구조입니다. RLS·RPC·트리거가 일부 기능(제보 검증 포인트, SOS 일일 한도, 신고 누적 제한 등)을 담당합니다.
- **Supabase Edge Functions**  
  - `moderate-user-content`: 현장 제보 텍스트·SOS 메모 **LLM 모더레이션**  
  - `verify-sos-photo`: SOS 첨부 사진 **비전 검증**  
  배포 시 Secrets(`GROQ_API_KEY`, Supabase URL/키 등)가 필요합니다.
- **클라이언트 UGC 안전**  
  SOS 사진: **EXIF 촬영 시각·스크린샷 태그** 검사(`sosPhotoIntegrity.ts`) 후에만 비전 API 호출.
- **지도 타일**: Carto CDN(브라우저 직접 요청).
- **이벤트·알림·연령대 군중 일부**: 여전히 **프론트 생성/더미**에 가까운 레이어가 있을 수 있음 — 기능별로 Supabase 연동 여부를 구분해 문서화할 것.

## 환경 변수

- **로컬 개발**: 베이스맵만 쓰면 필수 `.env` 없이도 동작할 수 있음.  
- **Supabase 연동 시**: Vite `import.meta.env`에 Supabase URL·Anon Key 등(프로젝트 `.env.example` 또는 내부 위키 참고).  
- **Edge AI**: Supabase 대시보드 **Secrets**에 `GROQ_API_KEY` 등 설정(클라이언트에 비밀키 두지 않음).
