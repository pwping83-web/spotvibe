## 1. 기능 요약 (한 줄)

**공공 대기질(미세먼지·통합대기) + (추후) 익명 인파 집계 + 사용자 프로필**을 입력으로, **실외·실내·활동 유형**에 대한 **짧은 방향 카드·알림**을 제공한다.

- 예: 「PM2.5 나쁨 → 오늘은 실외보다 실내」, 「혼잡 구역은 피하고 가벼운 산책」, 「등산 대신 ○○공원 30분」
- **스마트폰 카메라로 미세먼지를 직접 측정하지 않는다.** (현재 TF/BlazeFace는 **얼굴 모자이크** 전용)

## 2. 왜 지금이 아니라 «사용자 증가 후»인가

| 이유 | 설명 |
|------|------|
| **핵심 가치 우선** | SpotVibe 1차 차별화는 **지도·제보·SOS·시설 제보·매칭**이다. 대기질은 **재방문·부가 DAU** 축이지 코어가 아님 |
| **인파 신뢰** | 「인파 많은 곳 가지 마」는 **실제 익명 집계 또는 검증된 혼잡도** 없이 쓰면 **허위·불신** 리스크. 현재 지도 인파는 **더미/시뮬** 성격(`MapArea` AI_REC_POOL, `groqAiInsights` 가상 시나리오 명시) |
| **운영 부담** | 건강·감염 뉘앙스 문구 → **면책·민감정보·공공 API 출처** 정리 필요 |
| **비용 대비** | Edge 스케줄·DB·API 키·QA는 **사용자가 일상적으로 앱을 열 때** 효과가 큼. 초기에는 **카카오 심사·안정화·UGC/SOS**가 ROI가 높음 |

### 개발 착수 권장 조건 (게이트 — OR가 아니라 **대부분 충족** 후)

1. **DAU/MAU**가 내부 목표치 도달 (팀이 정한 수치 — 예: MAU 1만, DAU 1천 등 **별도 OKR 문서에 기록**)
2. **인파·혼잡**이 더미가 아닌 **서버 집계 1차** 또는 제3자 데이터 연동 로드맵 착수
3. **에어코리아(대기질) Open API** 활용신청·Secret 등록 완료 (`공공데이터-연계/` 절차)
4. **면책·위치정보** 문구 확정 (`위치정보-개인정보/01_위치정보-처리-문구.md` 보완)

> 게이트 수치는 본 문서에 고정하지 않는다. 착수 전 `03_기능-로드맵.md` 우선순위 표와 **팀 합의 1줄**을 메모해 둔다.

## 3. 현재 코드·문서에서 재사용할 것

| 구분 | 경로·패턴 | 역할 |

## 4. 단계별 구현 범위 (착수 시 로드맵)

### 4.1 1단 — MVP (사용자 증가 직후 권장)

- **데이터:** 에어코리아(또는 동급) Open API → Edge 캐시 → DB 스냅샷
- **위치:** 내 위치 / 탐색 중심(`explore_lat/lng`) → **가장 가까운 측정소** 매칭
- **출력:** 지도 또는 알림 탭 **1줄 브리핑** + **규칙 기반** 2~3문장 (Groq 없이 가능)
  - 예: PM2.5 등급 나쁨 → 「실외 장시간 비추 · 실내·가벼운 이동 권장」
- **제외:** 카메라 측정, 「코로나 때문에 ○○ 피하라」 단정, 인파 실시간 연동

### 4.2 2단 — AI·활동 연계

- Groq에 **구조화 JSON** (권장 활동 3 / 회피 2 / 근거 수치) — `groqAiInsights` 확장
- 기존 **야외운동·등산·공원** 시설 제보·`AI_REC` 태그와 **「등산 어때 → 대안」** 연결
- `MobilityProfile`·활동 태그 반영

### 4.3 3단 — 인파·혼잡·알림

- **서버 익명 집계** 또는 검증된 혼잡도와 결합 → 「붐비는 상권 회피」
- `notification_*` + 조건(나쁨 N시간 지속) → **푸시/알림 카드**
- 재난문자(`public_notices`)와 **링크만** 병행 (방역·의료 조언은 앱이 하지 않음)

### 4.4 4단 — 실증·지원사업 (선택)

- 환경부·지자체 **센서 실측·저감률**은 **앱 단독 증빙 불가** — 제3자 검증·IoT 파트너 분리
- 사업계획서 한 줄: `지원사업/02_사업계획서-공통-핵심항목.md` — 「공공 대기질·재난문자 연계 실외 활동 가이드」

- [ ] MVP 범위 서명: **대기질만 / 인파·감염 단정 없음**
- [ ] 노출 위치 결정: 지도 AI 패널 / 알림 탭 / 둘 다

## 6. 기술 초안 (착수 시 개발 참고)

```
[스케줄] pg_cron 또는 관리자 트리거
    → Edge sync-air-quality
    → air_quality_snapshots (station_id, pm25, pm10, grade, fetched_at, geo)

[클라]
    GPS / explore_lat,lng
    → RPC 또는 REST: nearest_station + latest_grade
    → 규칙 엔진 (서버 또는 Edge)
    → UI 카드 / (2단) Groq 카피

[금지]
    클라이언트에 공공 API 키
    카메라 기반 PM2.5 "측정"
    질병명·치료 조언 단정
```

**지도 UI:** `.cursor/rules/map-layout-safe-zones.mdc` — 하단 내비 `pb-[5.5rem]`, 우측 FAB `bottom-[15rem]` 이상.
