# 스팟바이브(SpotVibe) — 프로젝트 개요

## 한 줄 정의

**실시간·지역 맥락을 바탕으로 “지금 이 근처에서 무엇이 일어나는지”를 지도와 이벤트·알림으로 보여 주고, AI 스타일의 행동 지시로 이동·타이밍·무드를 돕는 모바일 웹 앱 프로토타입.**

브랜드 표기는 화면에서 **스팟바이브**로 통일하는 방향입니다.

---

## 컨셉

1. **핫스팟(Hotspot) + 바이브(Vibe)**  
   특정 구역(예: 홍대·연남 일대)에서 연령대·이벤트·군집(클러스터)으로 “활기”를 시각화합니다.

2. **탐색 중심**  
   사용자가 관심 지점을 바꾸면 히트맵·점 분포·이벤트 핀이 같은 기준으로 이동해, “내가 보는 중심”과 데이터가 맞춰집니다.

3. **행동 지시형 AI UI**  
   지도에서 클러스터 등을 탭하면 하단 시트가 열리고, 이동 수단(자차 / 킥보드 / 뚜벅이·따릉이)에 맞는 경로 안내 문구, 미니맵, 액션 카드, 외부 지도(네이버 검색 링크)를 제공합니다.

4. **마이 페이지 프로필**  
   선호 시간대, AI 추천 on/off, 카카오 연동 UI(모의), **이동 프로필(MobilityProfile)** 이 BottomSheet의 문구와 연결됩니다.

## 수익 구조 (기획 방향)

**핵심:** 이용자가 실제로 스크롤·탭하는 **이벤트 탭**에 **광고·유료 노출 구역**을 두는 모델입니다.  
당근마켓의 **지역(동네) 기반 광고**처럼, **“사람이 보는 곳”** 에 지역 행사·버스킹·페스티벌·매장 프로모션 등을 **유료 리스팅**으로 올리는 구조를 전제로 합니다.

| 요소 | 설명 |
|------|------|
| 인벤토리 | 이벤트 리스트 상단·피드 내 **파트너 슬롯**, 필터와 섞이지 않게 **광고 라벨**(예: 파트너·광고)로 식별 |
| 과금 단위 | 지역(반경·행정동)·기간·노출 순위(CPC/CPM/고정 월비 등은 사업·법무와 별도 확정) |
| 광고주 | 지역 상권, 공연·행사 주최, 브랜드 로드쇼, B2B 파트너 |
| 사용자 경험 | 자연스러운 “이벤트” 형태로 노출하되 **스폰서 표시**로 투명성 유지(표시광고법·플랫폼 정책 검토 필요) |

지도 탭의 핀·클러스터와 **연동한 유료 강조**는 2차 확장으로 둘 수 있습니다. 현재 UI의 **이벤트·파트너 표기**는 이 수익 모델을 염두에 둔 자리입니다.

## 주요 화면·기능 (현재 구현 기준)

| 영역 | 설명 |
|------|------|
| 로그인 | **Supabase Auth + 카카오 OAuth**; 모더레이션 상태에 따른 강제 로그아웃 등 |
| 지도 | Leaflet + Carto Dark Matter, 내 위치(GPS), 연령대 점(투명 군집), 이벤트 핀, 지역 프리셋 빠른 이동 |
| **AI 추천 버튼** | 지도 우하단 버튼 탭 → 마이 프로필 기반 **개인화 추천 카드**(30초 카운트다운, 다음 버튼으로 순환) |
| **AI 인도 말풍선** | 지도를 보는 중 **가끔(30~60초 간격)** 저절로 나타나는 「여기 어때? / 여기는?」 말풍선 + 화살표·점선 경로 |
| **심장 박동 오버레이** | MBTI·성별·활동 구역 원이 심장처럼 살짝 맥박 치는 애니메이션 |
| **내 위치 / 탐색 중심** | GPS 켜면 파란 점 + 「내 위치」, GPS 없으면 탐색 중심에 「탐색 중심」 라벨 |
| 이벤트 | 리스트·필터 칩·**파트너·광고** 표기(→ 유료 지역·행사 노출 구조와 연결) |
| 알림 | 타임라인형 알림, 무드 이미지 |
| 마이 | MBTI·혈액형·활동 태그·성별 인파·이동 프로필·선호 시간 설정 |
| **운영·문의** | 마이 탭 하단 — 개선 제안·광고·행사·기타를 메일 앱 연동으로 접수 |
| **헤더** | 스팟바이브 ← 실시간 시계 → 현재 탭 표시, 항상 최상단 고정 |

## 타깃 사용자(가정)

- 라이브·버스킹·야외 모임 등 **“지금 어디가 붐비는지”** 를 빠르게 보고 싶은 이용자.
- 대중교통·PM·자차 등 **이동 방식이 다른 사람**에게 맞춘 한 줄 안내를 원하는 이용자.

## 핵심 성장 이론 — 외로움·호기심 → SOS 문화 전환 루프

> 이 섹션은 앱의 **유기적 성장 메커니즘**을 설명합니다. 수익 모델이 없어도 문화적 관성으로 자생 성장할 수 있는 근거입니다.

### 1단계: 외로운 사람들이 지도를 켠다

갈 곳 없는 주말, 혼자인 밤, 무료한 퇴근길 — 이 앱의 첫 번째 사용자는 **사회적 외로움**을 느끼는 보통 사람입니다.  
"오늘 어디에 사람이 많지?" 라는 아주 작은 궁금증이 앱을 처음 켜게 만들고, 군집 밀도 지도는 이 호기심을 **시각적으로 만족**시킵니다. 능동적으로 나가지 않아도 **간접적 연결감**이 생기는 것이 중독의 씨앗입니다.

### 2단계: 지도 보는 습관이 생긴다

PRNG 기반 점 드리프트, MBTI 오버레이, AI 말풍선 인도 등은 지도를 **매번 조금씩 다르게** 만들어 관성적인 재방문을 유도합니다. 처음에는 하루 한 번, 이후 출퇴근 때마다, 결국 하루 여러 번 열게 되는 **DAU 중독 루프**가 형성됩니다.

### 3단계: SOS 마커가 지도에 뜬다

어느 날, 지도 위에 붉은 SOS 마커가 나타납니다. 이걸 본 사용자는 해결이 아닌 **순수한 호기심**으로 탭합니다.

### 4단계: 해결된 사례 사진이 보인다 (핵심)

SOS 마커를 탭하면 — 장애인이 계단에 고립됐던 사진, 이웃이 도와준 이후 올린 **AI 검증 현장 사진**이 붙어 있습니다. 텍스트로 설명하는 것과 달리, "누군가 실제로 나타나서 해결했다"는 시각적 증거는 심리적 장벽을 허무는 가장 강력한 도구입니다.

> "어, 이게 진짜 되네? 나도 저 근처에 있었으면 할 수 있었겠는데?"

이 한 문장이 **수동 관찰자를 능동 구조자로 전환**시키는 임계점입니다.

### 5단계: 선한 사람들이 먼저 움직인다

모든 사람이 움직일 필요는 없습니다. **종교인, 자원봉사 경험자, 간호·의료 종사자, 행동 지향적 사용자** — 이들은 SOS를 보고 주저 없이 나섭니다. 이 초기 히어로층이 앱에 **신뢰성·실효성**을 부여하고, 첫 사례들을 쌓습니다.

### 6단계: 인스타그램으로 전파된다

히어로 인증서 + 원터치 스토리 공유 → "#SpotVibe 건대입구역 휠체어 요정 등극" → 팔로워들이 앱을 다운로드합니다. 운영팀이 매주 실제 사례자를 인터뷰해 1분짜리 숏폼으로 제작하고, 언론이 받아쓰면 **"앱이 아니라 문화"** 가 됩니다.

### 7단계: 수익 없이도 자생한다

가입자가 많을수록 SOS 수신 가능한 이웃이 많아지고, 이웃이 많을수록 SOS가 실제로 해결되고, 해결될수록 더 많은 사람이 이 앱을 믿고 가입합니다. **네트워크 효과 + 사회적 신뢰 루프**가 완성되면, 마케팅 예산 없이도 DAU가 증가하는 구조입니다. 이후 지자체 SaaS 구독·이벤트 리스팅 광고는 이 트래픽을 기반으로 한 선택적 수익화입니다.

| 루프 단계 | 유저 심리 | 앱 내 기제 |
|-----------|-----------|------------|
| 첫 접속 | "어디 사람 많지?" (외로움·호기심) | 군집 밀도 지도, 다크모드 |
| 재방문 습관 | "오늘은 또 달라졌나?" (FOMO) | PRNG 드리프트, AI 말풍선 |
| SOS 목격 | "이게 뭐지?" (호기심) | 반경 3km 실시간 마커 |
| 사례 사진 | "진짜 됐네, 나도?" (사회적 증거) | AI 검증 해결 사진 노출 |
| 첫 구조 참여 | "해봤다!" (성취·정체성) | 히어로 인증서, 포인트 (로드맵) |
| SNS 공유 | "자랑하고 싶다" (과시·선행) | 원터치 인스타 공유 (로드맵) |
| 주변 가입 | "나도 해볼게" (동조) | 바이럴 루프 완성 |
